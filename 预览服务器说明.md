# 预览服务器使用说明

## 🔍 为什么每次改动后预览页面都打不开？

### 问题原因

1. **Python HTTP服务器不支持热重载**
   - 当您修改 `preview_server.py` 文件后，服务器仍在运行旧版本的代码
   - 修改的代码不会自动生效，需要手动重启服务器

2. **端口占用问题**
   - 旧的服务器进程仍在运行，占用端口 3003
   - 新的服务器无法启动，因为端口已被占用

3. **浏览器缓存**
   - 浏览器可能缓存了旧版本的页面内容
   - 即使服务器已更新，浏览器仍显示缓存的内容

## ✅ 解决方案

### 方案一：使用启动脚本（推荐）

双击运行 `start_preview.bat` 文件，脚本会自动：
1. 检查端口占用情况
2. 终止旧的服务器进程
3. 启动新的服务器

### 方案二：手动重启

1. **停止旧服务器**
   ```bash
   # 在运行服务器的终端中按 Ctrl+C
   ```

2. **终止占用端口的进程**
   ```bash
   # 查找占用端口的进程
   netstat -ano | findstr 3003
   
   # 终止进程（替换 <进程ID> 为实际的PID）
   taskkill /F /PID <进程ID>
   ```

3. **启动新服务器**
   ```bash
   python preview_server.py
   ```

### 方案三：刷新浏览器

修改 HTML 文件后：
1. **普通刷新**: 按 F5 或点击刷新按钮
2. **强制刷新**: 按 Ctrl+F5（清除缓存）
3. **硬刷新**: 按 Ctrl+Shift+R

## 💡 最佳实践

### 修改 HTML 文件
- ✅ 服务器已禁用缓存，修改后直接刷新浏览器即可
- ✅ 建议使用 Ctrl+F5 强制刷新，确保看到最新内容
- ✅ 修改会立即生效，无需重启服务器

### 修改 preview_server.py
- ❌ 需要重启服务器才能生效
- ✅ 使用 `start_preview.bat` 脚本自动重启
- ✅ 或者手动执行停止和启动命令

### 开发工作流

1. **启动服务器**（只需一次）
   ```bash
   # 双击运行
   start_preview.bat
   
   # 或手动启动
   python preview_server.py
   ```

2. **修改 HTML 文件**
   - 编辑 HTML 文件
   - 在浏览器中按 Ctrl+F5 刷新
   - 查看修改效果

3. **修改服务器代码**
   - 编辑 `preview_server.py`
   - 在终端中按 Ctrl+C 停止服务器
   - 重新运行 `python preview_server.py`
   - 或双击 `start_preview.bat`

## 🛠️ 故障排除

### 问题：端口被占用
**错误信息**: `OSError: [WinError 10048] 通常每个套接字地址(协议/网络地址/端口)只允许使用一次。`

**解决方案**:
```bash
# 1. 查找占用端口的进程
netstat -ano | findstr 3003

# 2. 终止进程
taskkill /F /PID <进程ID>

# 3. 重新启动服务器
python preview_server.py
```

### 问题：修改后看不到变化
**解决方案**:
1. 按 Ctrl+F5 强制刷新浏览器
2. 清除浏览器缓存
3. 检查文件是否保存成功
4. 确认修改的是正确的文件

### 问题：服务器无法启动
**可能原因**:
1. Python 未安装
2. preview_server.py 文件损坏
3. 端口被其他程序占用

**解决方案**:
1. 确认 Python 已安装: `python --version`
2. 检查文件是否存在: `dir preview_server.py`
3. 使用 `start_preview.bat` 自动处理端口冲突

## 📝 总结

- **修改 HTML**: 直接刷新浏览器即可
- **修改服务器代码**: 需要重启服务器
- **推荐做法**: 使用 `start_preview.bat` 脚本
- **强制刷新**: 按 Ctrl+F5 清除缓存

这样您就可以高效地进行开发和预览了！🎉
